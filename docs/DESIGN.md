# Design — RetroTech

> 제품/기획 의도, UX, 도메인 규칙, 주요 의사결정을 정리한 문서.
> 구조·구성은 [ARCHITECURE.md](./ARCHITECURE.md), 성능은 [PERFORMANCE.md](./PERFORMANCE.md) 참고.

## 제품 개요

**RetroTech** 는 *"기술의 역사를 자세히 설명하는"* 한국어 팟캐스트다. 특정 기술이 어떤 배경에서 등장하고 발전했는지, 또 왜 어떤 기술은 사라졌는지를 다룬다 (예: 2g화 "VCS: SourceForge").

이 웹사이트는 그 팟캐스트의 **허브**다. 역할은 두 가지다.

1. **구독 유도** — 청취자를 각 팟캐스트 플랫폼(Apple/Spotify/YouTube/Google/RSS)으로 보낸다. 사이트 자체가 주 재생 환경은 아니다.
2. **쇼노트/레퍼런스 제공** — 각 에피소드에서 다룬 내용과 출처 링크를 정리해 둔다. 역사·고증 콘텐츠라 레퍼런스 목록이 길고 중요하다(2g화는 참고링크 약 60개).

## 타깃 / 톤

- **한국어 사용자.** `<html lang="ko">`, 한글 우선 폰트 스택, 모든 콘텐츠 한국어.
- 1인 제작(호스트: **Outsider**). 푸터에 호스트 프로필·SNS·후원(GitHub Sponsors) 링크 노출.
- 미니멀한 블로그형 UI. Nextra 블로그 테마의 기본 외형을 거의 그대로 사용하고 군더더기를 더하지 않는다.

## 콘텐츠 구조 & 도메인 규칙

- **에피소드 = MDX 파일 1개.** 메타데이터는 프론트매터, 쇼노트는 본문. (스키마: [ARCHITECURE.md](./ARCHITECURE.md#라우팅--콘텐츠-모델))
- **회차 번호 체계:** `시즌 + 알파벳` (`1a`…`1n`, `2a`…). `0` 은 소개 회차. 날짜형 슬러그(`250127-breaks`)는 정규 시즌 외 "쉬어가는" 회차.
- **에피소드 페이지 구성(관례):**
  1. 제목(`# 2g. ...`)
  2. 도입 요약 단락(프론트매터 `description` 과 동일 내용)
  3. `<Badges/>` — 그 회차의 플랫폼별 구독/청취 링크
  4. `#### 레퍼런스:` + `<div className="refs">` 안의 링크 목록
  5. `#### 배경음악` 출처 표기(Uppbeat 라이선스 등)
- **레퍼런스는 1급 콘텐츠다.** 역사 고증 특성상 출처(위키, 웹아카이브, 인터뷰, SEC 문서 등)를 충실히 남긴다. `.refs` 는 본문보다 작은 글씨로 처리해 밀도를 높인다.

## 배포 채널 & 구독 (핵심 UX)

사이트의 1차 전환 목표는 "플랫폼 구독"이다. 이를 위한 장치:

- **`components/Badges.tsx`** — 재사용 가능한 구독 배지 묶음. 기본값은 쇼 전체 링크(채널), 에피소드에서는 props 로 해당 회차의 딥링크를 넘긴다.
  - Apple Podcasts, YouTube, Spotify 는 항상 노출.
  - **Google Podcasts 는 선택적.** `google` prop 이 있으면 Google 배지를, 없으면 대신 **RSS 배지**(`/feed.xml`)를 노출한다. (Google Podcasts 서비스 종료 이후 RSS 로 대체하는 의도로 해석됨.)
- **RSS 피드가 정식 배포물.** `feed.xml` 은 iTunes 팟캐스트 규격으로 생성되어 Apple 등에 등록되는 실제 피드다. 푸터·`<head>`·홈에서 모두 `/feed.xml` 로 접근 가능.
- 오디오(mp3)는 `retrotech-episodes.outsider.dev` 에 별도 호스팅하고 프론트매터 `enclosure` 로 연결한다(사이트와 스토리지 분리).

## 디자인 결정 & 그 이유

| 결정 | 이유 / 배경 |
| --- | --- |
| 정적 익스포트(서버 없음) | 콘텐츠가 정적이고 업데이트 빈도가 낮음. 호스팅 단순화·비용 최소화. |
| Nextra 블로그 테마 채택 | 글 중심 콘텐츠에 적합. 목록/다크모드/타이포를 직접 만들 필요 없음. |
| MDX + 프론트매터 단일 소스 | 같은 파일에서 웹 페이지와 RSS 아이템을 동시 생성(중복 방지). |
| 사이트 내 재생기 미제공 | 1차 목표가 "플랫폼 구독 유도"라 청취는 외부 플랫폼으로 보냄. |
| 이미지에 `next/image` 사용하되 akamai 로더 | 정적 익스포트 제약. 실제로는 최적화 없이 원본 전송(주의: [PERFORMANCE.md](./PERFORMANCE.md)). |
